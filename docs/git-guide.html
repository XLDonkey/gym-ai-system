1. What Is GitHub?

GitHub is where we store and version-control all our code. Think of it like Google Drive for code — every change is tracked, nothing is ever permanently lost, and multiple people can work on it without overwriting each other.

Our GitHub Account
github.com/XLDonkey

This is the XL Fitness developer account. All our repos (projects) live here.

Key Concepts

TermWhat it means
RepoA project folder. We'll have one per system (pose tracking, face tracking, etc.)
CommitA saved snapshot. Every time Donkey makes a change, it's committed with a message explaining what changed.
BranchA parallel version of the code. We develop on dev, and only move to main when it works.
PushUploading commits from the Mac to GitHub.
GitHub PagesFree hosting — anything on main is automatically live at our URL.

4. Long-Term Repo Structure

Right now we have one repo. As the system grows, each major component gets its own repo. This keeps things clean and lets different parts move at different speeds.

RepoWhat it doesStatus
xlf-pose-trackingRep counting, form scoring, exercise detection✅ Active
xlf-face-trackingFace ID, member recognition, occupancy detection🔜 Phase 3
xlf-weight-trackingWeight stack reading, plate detection via camera🔜 Phase 4
xlf-member-dataMember profiles, session history, API layer🔜 Phase 3
xlf-demoStakeholder demo site — pulls from other repos🔜 Phase 2
xlf-infraInfrastructure configs, deployment scripts, network setup🔜 Phase 3
Why separate repos?

If face tracking is broken, it doesn't affect the rep counter. Each repo deploys independently. Each has its own demo URL, its own training data, its own README. Clean separation = less risk.

  3. Message Donkey — I can roll back to any previous working version instantly
  4. Nothing is ever permanently lost — every change is saved in git history
