⚠️ Key Insight Before Anything Else

Only 1 of the 3 cameras per machine goes in or near the ceiling. The other 2 go on walls, columns, or the machine itself. Drilling into the ceiling is a Phase 2 decision — Phase 1 can be done without touching the ceiling at all.

2. What the AI Learns — Training States

The AI Overseer needs to recognise 7 different states. Each state needs training data — real footage of that state happening. Here's what each one is and how to film it.

State 1
⬜ No User Present
Empty machine. No one in frame at all. Tells the system the machine is free. Film: just let the camera run when the machine is empty.
State 2
🚶 User Present, Not Engaged
Someone near the machine but not using it — walking past, adjusting the seat, talking. Film: walk near the machine, stand around it without sitting.
State 3
🪑 Seated, Engaged
Person seated, gripping bar, ready to pull. This is the TRIGGER — when AI sees this it activates rep counting. Film: sit down, grip bar, hold still 3–5 seconds.
State 4
✅ Good Rep
Full pull-down, controlled return, proper form. Full range of motion (bar to chest level). Film: do proper lat pulldowns, full ROM, controlled tempo, back straight.
State 5
⚠️ Bad Rep
Partial pull, excessive swing/lean-back, rushed. Film: deliberately bad reps — lean way back and swing, only pull halfway, rush the movement.
State 6
⚡ False Movement
Arm movements that aren't reps — adjusting grip, reaching for towel, talking with arms, scratching head. Film: sit at machine and make various non-rep arm movements.
State 7
⏱ Rest Between Sets
Seated at machine, not pulling. Resting, checking phone, breathing. Tracking this gives rest time data and prevents false counting. Film: sit at machine between sets and rest naturally for 60–120 seconds.

3. Hardware — Every Component Explained

The Cameras

PoE — Power over Ethernet

The most important thing to understand about these cameras. One cable does two things — it carries the video data from the camera to the server, AND it carries power to the camera. You do not need a separate power outlet near each camera. One CAT6 cable to each camera = done.

Why 1080p and not 4K?

MoveNet (the pose AI) internally resizes every frame to 192×192 pixels before processing. Sending 4K video to get 192×192 analysis is like taking a photo with a $5,000 camera to print a thumbnail. 1080p is perfect — anything higher is wasted bandwidth and storage.

Why 30fps?

Rep counting requires smooth motion tracking. At 15fps, fast movements blur between frames and the elbow angle jumps around. At 30fps, the angle changes smoothly and accurately. 60fps adds no benefit for this use case.

WDR — Wide Dynamic Range

XL Fitness has floor-to-ceiling windows on multiple walls. The contrast between the bright outside and the dark gym interior is extreme. WDR handles this by effectively taking two exposures simultaneously — one for the bright areas, one for the dark — and combining them. Without WDR, users near windows become silhouettes. Rated in dB — we want 120dB minimum, 130dB ideal.

ColorVu vs IR

Standard IR cameras give black-and-white vision in low light (the red glow you see in security cameras). ColorVu cameras use a highly sensitive sensor and a white light LED to give full-colour imaging at near-total darkness (0.0005 lux). XL Fitness has a dark, neon-lit interior — ColorVu is the right choice. The neon lighting actually helps.

The Edge Server — Mac Mini M4 Pro

This is the brain. All camera video goes into this box. It runs all the AI models simultaneously, works out what's happening at each machine, and sends results to the database. It sits in a small comms cabinet — members never see it.

Why Mac Mini and not a Windows PC?

  • Apple Silicon (M4) has a dedicated Neural Engine — 38 trillion operations per second specifically designed for AI workloads
  • Runs completely silently — no fans in a gym environment
  • Compact — 197×197×35mm, fits in any comms cabinet
  • Extremely power efficient — 20–40W typical load vs 200–500W for equivalent PC
  • macOS is extremely stable for always-on server use
  • Runs Python, TensorFlow, FastAPI natively

The Ceiling Void (Plenum)

The space between the black suspended tile ceiling you can see and the actual concrete slab roof above it. All cables, pipes and ducts run through here. The decorative LED halo lights at XL Fitness are already wired through this space — camera cables will do exactly the same. A cabler pops a tile, threads the cable, replaces the tile. Nothing visible.

✅ No Ceiling Tiles Get Drilled

Tiles are pushed up and slid aside to access the plenum — they are NOT drilled through. The cable runs through the space above, comes down at the camera location, and the tile sits back in place. Camera bracket clips to the metal T-bar grid rails, or mounts on a column — tiles are untouched.

4. How the Software Works

Layer 1 — The Camera Feed

Each camera streams video continuously over the local network. Think of it as a permanent live video call from each camera to the Mac Mini. The video is compressed using H.265 (the same format used by Netflix) to keep file sizes manageable without losing quality.

Layer 2 — The AI Models

Pose Estimation (Donkey's code — gym-ai-pose)

  1. MoveNet receives each video frame from Camera 2
  2. It identifies 17 body landmarks: nose, shoulders, elbows, wrists, hips, knees, ankles
  3. The angle between shoulder → elbow → wrist is calculated (the elbow angle)
  4. This happens 30 times per second
  5. The LSTM model analyses the sequence of angles over time: "is this pattern a rep? Good or bad?"
  6. Output: rep_completed, quality: good, confidence: 0.91

Weight Detection (Manus's code — gym-ai-weight)

  1. Camera 3 feeds into the weight detection model
  2. A pre-configured bounding box isolates the weight stack in the frame
  3. Colour detection identifies tape on plate spines (White=5kg, Yellow=10kg, Blue=20kg)
  4. Adds up all detected plates above the selector pin
  5. Output: weight_detected: 50kg

Face ID (Phase 3)

  1. Camera 1 captures the member as they sit at the machine
  2. The Re-ID model compares their face/body against known member embeddings in the database
  3. If confidence > 95%: member_identified: Matt McArdle
  4. If confidence < 95%: tablet shows "Please tap your card to begin"
  5. Card tap = 100% certainty = that data labels the face footage for future retraining

Layer 3 — The Orchestration Server (Manus building)

This is the traffic controller. All the AI models send their results here as JSON events. The server combines everything: "Matt is at machine 7, Camera 3 says he loaded 50kg, Camera 2 says he just did a good rep." It maintains the state machine for each machine in the gym.

State machine per machine:
  no_user_present
       ↓ (face ID or card tap)
  user_identified
       ↓ (Camera 2 detects seated pose)
  user_seated_engaged  ← rep counter ACTIVATES here
       ↓ (rep detected)
  rep_completed (×N)
       ↓ (no movement for 30s)
  rest_period  ← rest timer starts
       ↓ (grips bar again)
  user_seated_engaged
       ↓ (user leaves frame)
  session_complete  ← writes to database

Layer 4 — The Database (Supabase)

All data lives permanently in Supabase — a cloud-hosted PostgreSQL database. Every rep, every set, every session is stored. If the internet goes down, the server buffers locally and syncs when reconnected. Members can access their data from anywhere.

Tables:
  members      (id, name, gymmaster_id, face_embedding)
  sessions     (id, member_id, machine_id, start_time, end_time)
  sets         (id, session_id, reps, good_reps, weight_kg, form_score, rest_time_s)
  events       (id, session_id, timestamp, event_type, payload)

Layer 5 — The Outputs

  • Tablet display — receives webhook → shows rep count live → "Set complete: 10 reps @ 50kg, form 84/100"
  • Leaderboard — Power BI (interim) or custom dashboard queries Supabase → updates live
  • Member profile — personal bests, session history, form trends over time
  • Notifications (future) — "New personal best! 12 reps @ 55kg"

6. Rest Time Tracking

How It Works

Rest time is tracked automatically by the state machine. When the AI stops detecting active rep movement while the user remains at the machine, the rest timer starts. When they grip the bar again, rest time is logged and set 2 begins.

State transitionWhat happens
Rep completes → no movement for 15sState → rest_period. Timer starts.
User grips barState → seated_engaged. Rest time logged (e.g. 92s).
User stands and leavesState → session_complete. Final rest time logged.

Why Rest Time Matters

  • Rest time is one of the most important training variables — but currently no gym tracks it at all
  • Most members have no idea how long they're actually resting between sets
  • Strength training research recommends 2–4 minutes for compound lifts. Most people rest 5–10 minutes.
  • The AI can flag: "You averaged 6:20 rest between sets. Try reducing to 3:00 for better stimulus."
  • Rest time data is valuable for personal trainers advising members remotely
  • Future: personalised rest recommendations based on exercise, weight, and member goal
💡 Competitive Advantage

No gym in the world automatically tracks rest time per set. This is a genuinely novel data point that XL Fitness will own. It becomes a selling point: "We're the only gym that tracks not just your reps, but your actual rest periods — and optimises your training accordingly."

7. Phase 1 Physical Setup Checklist

Getting the prototype running at the lat pulldown. No ceiling work required. No electrician required. Just a clamp and a phone/tablet.

Before You Start

  • Identify the purple structural column nearest to the lat pulldown machine
  • Confirm it faces the machine side-on — you should be able to see the user's full profile (not front-on)
  • Check there are no windows directly behind where the user sits (from the camera's perspective)
  • Measure approximate distance: column to machine seat should be 2–3m

Get These Items (~$50–80 total)

  • SmallRig magic arm with C-clamp (~$50, Amazon AU) OR generic phone tripod clamp (~$20, Jaycar)
  • Alternatively: any portable tripod + phone/tablet mount

Mount the Camera

  • Clamp the arm to the purple column at approximately 1.3m height
  • Position the phone/tablet side-on to the machine (90° to the user)
  • The user's full upper body must be visible: head to hips at minimum
  • The elbow should be clearly visible in frame when pulling the bar
  • Windows should be BEHIND the camera, not behind the user

Test the System

  • Open Chrome on the tablet
  • Go to: xldonkey.github.io/gym-ai-system/pose/alpha.html
  • Allow camera access
  • Sit at the machine — green skeleton should appear on your body
  • Check: elbow angle number changes as you move your arms
  • Pull the bar down fully — angle should drop below 90°
  • Release — angle should rise above 130°. Rep counter should increment.
  • If skeleton is unclear or silhouetted: move camera to opposite side so windows are behind it

Film Training Data

  • Use a separate device (GoPro, iPhone, second phone) on a tripod — same side-on position
  • Record Clip 1: Good reps — 10 minutes, proper form, full ROM
  • Record Clip 2: Bad reps — 5 minutes, deliberately poor form (swing, lean back, partial)
  • Record Clip 3: False movements — 5 minutes, sit at machine and make non-rep arm movements
  • Upload all 3 clips to Google Drive
  • Open the labeler: xldonkey.github.io/gym-ai-system/pose/label.html
  • Load each video, label sections (S=start, G=good, B=bad, F=false, E=seated, N=empty)
  • Export JSON from labeler
  • Send Donkey: Google Drive links + JSON files → Donkey retrains → accuracy improves
✅ What Success Looks Like at Phase 1

A tablet clamped to the purple column, skeleton visible on the user, rep counter incrementing accurately for good reps, not incrementing for bad form or false movements. That's it. That's the Phase 1 prototype. Everything else — weight tracking, face ID, server integration — builds on top of this working foundation.
