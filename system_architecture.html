Prepared By Manus AI
Date March 2026
Version 1.0 — Stakeholder Brief
Focus Phase 3: Multi-Camera Per Machine

1. Executive Summary

This document outlines a scalable system architecture for the XLFitness AI platform, designed to integrate multiple camera inputs per machine for comprehensive workout tracking. The proposed architecture addresses the challenges of person identification, rep counting, and weight detection by assigning one primary job to each camera. This separation of concerns simplifies the AI models, increases accuracy, and creates a robust, extensible system. A central server, running on the planned Mac Mini M4 edge device, will orchestrate the data flow, run the AI models, and persist session data to the Supabase cloud database.

2. The Multi-Camera Per Machine Model

To achieve reliable tracking from overhead, a single camera is insufficient. A three-camera setup per machine is recommended, with each camera positioned and optimized for a single task. This modular approach allows for independent development and upgrades of each component.

Camera Position Primary Job AI Model / Technique Status
Camera 1: User ID Overhead, centred on user area Identify the member at the machine Face Recognition or Person Re-ID To be built
Camera 2: Rep Counter Overhead/angled, clear view of body Track joint angles and count reps Pose Estimation (MoveNet/BlazePose) + LSTM In progress (XLDonkey)
Camera 3: Weight Detector Angled down at weight horn/stack Read the loaded weight at rest Colour Detection + Bounding Box POC built (Manus)

4. Key Components & Recommendations

Recommendation 1: Physical Markers are Essential

Purely vision-based solutions for weight detection with all-black plates are unreliable. Simple physical markers provide the ground truth the AI needs.

  • Plates: Apply coloured rubber electrical tape to the outer spine/edge of each plate. This ensures visibility from a side/angled camera view, even when stacked.
  • Colour Code: White = 5kg, Yellow = 10kg, Blue = 20kg.

Recommendation 2: Bounding Box for Weight Detection

The most robust way to isolate the active weight horn is a pre-configured bounding box, not a physical marker on the horn itself.

  • Configuration: A one-time setup per machine where an admin draws a box on the camera feed around the active horn's resting position.
  • Storage: The box coordinates (x, y, width, height) are saved to a machine-specific JSON file (e.g., `machine_configs/lat_pulldown.json`).
  • Execution: The weight detection AI only processes pixels inside this box, ignoring storage horns, the floor, and other visual noise.

Recommendation 3: Central Server for Orchestration

A custom server application running on the Mac Mini is the hub that connects everything. It is responsible for:

  • Receiving Streams: Ingesting video from all cameras.
  • Running Models: Executing the appropriate AI model for each camera's job.
  • State Management: Knowing when a machine is in use vs. at rest, and triggering the correct AI (e.g., run weight detection only at rest).
  • Data Aggregation: Combining the outputs (User ID + Reps + Weight) into a single session record.
  • Database Sync: Pushing completed session records to Supabase.

5. Development & Integration Plan

This architecture allows for parallel development. The following table outlines the responsibilities and integration points.

Component Lead Developer Key Task Integration Point
Rep Counter (LSTM) XLDonkey AI Train and deploy the overhead pose estimation and LSTM model. Server will call this model with video from Camera 2. Model returns rep count and quality.
Weight Detector Manus AI Refine the colour detection model for spine-reading within a bounding box. Server will call this model with video from Camera 3. Model returns total weight.
User ID TBD Implement face recognition or person re-ID model. Server will call this model with video from Camera 1. Model returns member ID.
Central Server & DB Manus AI Build the server application, state logic, and Supabase integration. The core orchestrator that connects all other components.

7. End Vision: The AI Gym Floor

The ultimate goal is to create a single, unified AI system that perceives the entire gym floor as a human would. This goes beyond discrete per-machine tracking and moves towards a holistic, ambient intelligence. In this vision, the network of cameras (potentially 30-40 across the gym) acts as a single set of eyes, feeding into a central AI brain.

This "AI Brain" would not just count reps on one machine, but understand the entire context of the gym floor. It would know:

  • Member Journeys: Tracking a member from the entrance, to the locker room, to their first machine, and through their entire workout circuit.
  • Machine Occupancy & Queues: Identifying not just that a machine is in use, but who is using it and who might be waiting nearby.
  • Cross-Machine Insights: Understanding that a member just finished a set of bench press and is now moving to the lat pulldown, allowing for intelligent workout recommendations.
  • Safety & Security: Detecting falls, incorrect machine usage, or unauthorized access.

This end vision moves from a collection of individual trackers to a true ambient intelligence platform for the entire fitness facility, providing unparalleled data and insights for both members and gym owners. The multi-camera-per-machine architecture is the foundational first step towards this ultimate goal.
